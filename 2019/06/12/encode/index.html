为什么需要编码？能不能不编码？

解释这个问题，必须要回答计算机是如何表示我们人类能够理解符号的，这些符号也就是我们人类使用的语言。由于人类的语言太多，表示这些语言的符号太多，无法用计算机的基本存储单元——字节（byte）来表示，因而必须要经过拆分或一些翻译工作，才能让计算机理解我们的语言。

把计算机能够理解的语言假定为英语，其他语言要能够在计算机中使用，必须经过一次翻译，把他翻译成英语。这个翻译的过程就是编码。所以可以想象，只要是不说英语的国家，要使用计算机都必须经过编码。这看起有些霸道，但就是现状。职业和我国在大力推广汉语一样，希望其他国家都说汉语，以后其它语言就被翻译成汉语，我们可以吧计算机中存储的信息的最小单位改成汉字，这样就不存在编码的问题了。

总的来说，编码存在的原因一下几条:

  • 在计算机中存储信息的最小单元是1个字节，即8 bit，所以能表示的字符范围是 0 ~ 255个
  • 人类标识的符号太多，无法用一个字节完全表示。比如最常见的汉字都有3000多个，很明显超过了一个字节的承受范围

如何“翻译”

各种语言交流，经过翻译是必要的，那么如何来翻译呢？
在计算机中提供了多种翻译方式，常见的有ASC、ISO-8859-1、GB2312、GBK、UTF-8、UTF-16等，他们都可以被看做是字典。
目前编码格式很多，如GB2312、GBK、UTF-8、UTF-16都可以标表示汉字，那我们到底选择哪种格式的编码来存储汉字呢？这就要考虑到其他因素了。例如，是存储空间重要还是编码效率重要。

存储效率

不同的编码占用的字节数不一样，头部常用的字符可能只占用1个字节，一些生僻的不常用或者几乎用不到的字符可能排到需要占用3个字节，如何分配内存存储这个鞋字节呢？我们可以为所有字符都分配三个字节的内存，也可以为编号小的字符分配一个字节或者两个字节的内存，而为编号大的字符分配三个字节的内存。

方案1：为每个字符分配固定长度的内存
一种方案是为每个字符分配固定长度的内存，并且这块内存要足够大，可以容纳下所有的字符编号。这种方案最简单，直接将字符编号放入内存中即可，不需要任何转换，并且以后在字符串中定位字符、修改字符都非常容易。

目前的 Unicode 已经收录了上百万的字符，至少需要三个字节才能容纳下所有的字符编号。假设字符串”A3中￥”的 Unicode 编码值（十六进制形式）分别是 2A、31、DA49、BB672C，那么它们在内存中的存储形式为：

在几乎所有的字符集中，常用字符的编号往往比较小，罕见字符的编号往往比较大，包括 Unicode 在内。
A和3是 ASCII 编码中的字符，Unicode 为了兼容 ASCII，在设计时刻意保留了原来 ASCII 中字符的编号，所以英文字母和阿拉伯数字在 Unicode 中的编号都非常小，用一个字节足以容纳。中是一个汉字，编号比较大，一般要用两个字节才能容纳。￥可以看做是一个极其少见，或者只有极少数地区才会使用到的字符，这样的字符编号往往比较大，有时候需要三个字节才能容纳。

上图中带灰色背景的字节是没有用到的字节，它们就是被浪费掉的一部分内存空间，这就是用固定长度的内存来存储字符编号的缺点：常用字符的编号都比较小，这种方案会浪费很多内存空间，对于以英文为主的国家，比如美国、加拿大、英国等，内存利用率甚至会低于 50%。

这样虽然没有了空闲字节，不浪费任何内存空间了，但是又出现新的问题了：如果我不告诉你，你怎么知道2A表示一个字符，而不是2A31或者2A31DA才表示一个字符呢？后面的字符也有类似的问题。

对于第一种方案，每个字符占用的字节数是固定的，很容易区分各个字符；而这种方案，不同的字符占用的字节数不同，字符之间也没有特殊的标记，计算机是无法定位字符的。

这种方案还需要改进，必须让不同的字符编码有不同的特征，并且字符处理程序也需要调整，要根据这些特征去识别不同的字符。

要想让不同的字符编码有不同的特征，可以从两个方面下手：
1) 一是从字符集本身下手，在设计字符集时，刻意让不同的字符编号有不同的特征。

例如，对于编号较小的、用一个字节足以容纳的字符，我们就可以规定这个字符编号的最高位（Bit）必须是 0；对于编号较大的、要用两个字节存储的字符，我们就可以规定这个字符编号的高字节的最高位必须是 1，低字节的最高位必须是 0；对于编号更大的、需要三个字节存储的字符，我们就可以规定这个字符编号的所有字节的最高位都必须是 1。

程序在定位字符时，从前往后依次扫描，如果发现当前字节的最高位是 0，那么就把这一个字节作为一个字符编号。如果发现当前字节的最高位是 1，那么就继续往后扫描，如果后续字节的最高位是 0，那么就把这两个字节作为一个字符编号；如果后续字节的最高位是 1，那么就把挨着的三个字节作为一个字符编号。

这种方案的缺点很明显，它会导致字符集不连续，中间留出大量空白区域，这些空白区域不能定义任何字符。

2) 二是从字符编号下手，可以设计一种转换方案，字符编号在存储之前先转换为有特征的、容易定位的编号，读取时再按照相反的过程转换成字符本来的编号。

相比第一种方案，这种方案有缺点也有优点：

  • 缺点就是多了转换过程，字符在存储和读取时要经过转换，效率低；
  • 优点就是在制定字符集时不用考虑存储的问题，可以任意排布字符。

常用字符编码

1.ASCII 码
学过计算机的都知道ASCII码，总共128个，用1个字节的低7位标识，0-31是控制字符如换行、回车、删除等，32-126是打印字符，可以通过键盘输入并且能够显示出来的。

ISO-8859-1
128个字符显然是不够用的，于是ISO组织ASCII码的基础上又定制了一系列标准来扩展ASCIi编码，他们是ISO-8859-1至ISO-8859-15，其中ISO-8859-1涵盖了大多西欧语言字符，所以应用的最广泛。ISO-8859-1仍然是单字节编码，他总共能表示256个字符。

GB2312
GB2312全称是《信息技术 中文编码字符集》，他是双字节编码，总的编码范围是A1-F7，其中A1-A9是符号区，总共包含682个字符；B0-F7是汉字去，包含6763个汉字。

GBK
GBK裙撑是《汉字内码扩展规范》，是国家技术监督局为Windows 95所定制的新的汉字内码规范，他的出现是为了扩展GB2312，并加入了更多的汉字。他的编码范围是8140~FEFE（去掉XX7F），总共有23940个码位，他能标识21003个汉字，他的编码是和GB2312兼容的，也就是说用GB2312编码的韩总也可以用GBK来解码，并且不会乱码。

GB18030
GB18030全称是《信息技术 中文编码字符集》，是我国的强制标准，他可能是单字节、双字节或者四字节编码，他的编码与GB2312兼容，虽然是国家保准，但是在实际应用中使用的并不广泛。

UTF-16
说到UTF必须提到Unicode，ISO师徒创建一个全新的超语言字典，世界上所有的语言都可以通过这个字典来互相翻译。可想而知这个字典是有多么复杂。关于Unicode的详细管饭可以参考相应文档。Unicode是Java和XML的基础，下面详细介绍Unicode的计算机中的存储形式。

UTF-16具体定义了Unicode字符在计算机中的存储方法。UTF-16用两个字节来表示Unicode的转化格式，它采用订场的标识方法，即不论什么字符都可以用两个字节表示。两个字节是16个bit，所以叫UTF-16。UTF-16表示字符非常方便，每两个字节表示一个字符，这就大大简化了字符串操作，这也是UTF-16作为内存的字符存储格式的一个重要原因。

UTF-8
UTF-16采用统一的量字节标识一个字符，虽然在标识上非常简单、方便，但是也有很大缺点，有很大部分字符用的一个字节就可以标识的现在要用两个字节表示，存储空间放大了一倍，在现在网络带宽还非常有限的情况下，这样会增大网络传输的流量，而且也没有必要。而UTF-8采用的一种变长的技术，每个编码区域有不同的字码长度。不同类型的字符可以由1-6个字节组成。

  • 如果是一个字节，最高位（第八位）为0，则表示这个是一个ACII字符（00~7F），所以是兼容ASCII的
  • 如果是一个字节，以11开头，则连续的1的个数暗示这个字符的字节数，例如：110××××× 代表的是双字节UTF-8字符的首字节
  • 如果是1个字节，以10开始，标识他不是首字节，则需要向前查找才能得到当前字符的首字节

2.按照GB2312编码

可以看出，前5个字符经过编码仍然是5个字节，而汉字责备编码成了双字节，GB231只支持6763个汉字，所以不是所有的汉字都能使用GB2312编码。

3.按照GBK编码

GBK与GB2312的编码结果是一样的，有次得出GBK编码是兼容GB2312编码的，他们 的拜纳姆算法也是一样的。不同的是，他们的码表长度不一样，GBK包含的汉字字符更多。所以只要是经过GB2312编码的汉字都是可以用GBK进行解码的，反之则不然。

4.按照UTF-16编码

用UTF-16编码将char数组放大了1倍，单字节范围内的字符在高位补0变成两个字节，中文字符也变成了两个字节。从UTF-16编码规则来看，仅仅是将字符的高位和低位进行拆分变成两个字节，特点是编码效率非常高，规则很简单。

5.按照UTF-8编码

UTF-8单字节范围的字符仍然用的是1个字节标识，对汉字采用3个字节表示。

对几种编码方式比较

对中文字符串，上述几种编码方式都能处理

  • GB2312与GBK贵吗规则类似，但是GBK范围更大，他能处理所有的汉字字符，所以这两个比较的话应选择GBK
  • UTF-16与UTF-8都是处理Unicode编码，他们的编码规则不同。相对来说，UTF-16的编码效率较高，从字符到字节的相互转换简单，进行字符串操作也更好。它使用在本地磁盘和内存直接使用，可以进行字符和字节直接快速切换，如Java的内存编码就是采用UTF-16编码。但是他不适合网络之间传输，因为网络传输容易损坏字节流，一旦字节流损坏很难恢复，所以相对比较而言UTF-8更适合网络传输。
  • UTF-8对ASCII字符采用单字节存储，另外单个字符损坏也不会影响后续字符，在编码效率上结余GBK和UTF-16之间，所以UTF-8在编码效率和安全性上做了平衡，是理想的中文编码方式。
